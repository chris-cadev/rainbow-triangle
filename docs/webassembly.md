# WebAssembly Build — Reference

Target: compile the game to run in a browser via WASM + Emscripten.

---

## Prerequisites

- Emscripten SDK installed at `~/emsdk`
- Raylib built for Web (see below)
- Game code is platform-independent (uses raylib, no Win32 API calls)

---

## Emscripten Environment Setup

```powershell
# One-time activation (run each new terminal session):
C:\Users\chris\emsdk\emsdk_env.ps1

# Or add to your PowerShell profile for automatic setup:
# Add-Content $PROFILE "C:\Users\chris\emsdk\emsdk_env.ps1"
```

Verify it worked:
```powershell
emcc --version
```

---

## Build Raylib for Web

Only needed once (or when raylib updates):

```powershell
cd raylib

# Configure for Web
mkdir build_wasm
cd build_wasm
emcmake cmake .. -DBUILD_SHARED_LIBS=OFF -DPLATFORM=Web -DCMAKE_INSTALL_PREFIX=..\install_wasm

# Build
emmake make
emmake make install
cd ..\..
```

This produces:
- `raylib/install_wasm/lib/libraylib.a` — static lib for WASM
- `raylib/install_wasm/include/raylib.h` — headers

---

## Compile Game for WASM

Basic command:

```powershell
emcc -o index.html src/main.cpp `
    -Iraylib/install_wasm/include `
    -Lraylib/install_wasm/lib `
    -lraylib `
    -O2 `
    -s TOTAL_MEMORY=67108864 `
    -s FORCE_FILESYSTEM=1 `
    --preload-file assets
```

### Output files

| File | Description |
|------|-------------|
| `index.html` | Shell page with canvas, loads .js + .wasm |
| `index.js` | Emscripten glue code (loader + runtime) |
| `index.wasm` | Compiled WebAssembly binary |

### Key Emscripten flags

| Flag | What it does |
|------|-------------|
| `-o index.html` | Output format: HTML + JS + WASM |
| `-O2` | Optimize (use `-O3` for release, `-O0` for debugging) |
| `-s TOTAL_MEMORY=67108864` | 64MB initial memory (increase if needed) |
| `-s FORCE_FILESYSTEM=1` | Enables virtual file system for assets |
| `--preload-file assets` | Packs `assets/` into the .data file loaded at startup |
| `--shell-file shell.html` | (optional) Custom HTML shell template |
| `-s USE_GLFW=3` | (if using GLFW instead of raylib's internal platform) |

### Debug build

```powershell
emcc -o index.html src/main.cpp `
    -Iraylib/install_wasm/include `
    -Lraylib/install_wasm/lib `
    -lraylib `
    -O0 -g `
    -s TOTAL_MEMORY=67108864 `
    -s FORCE_FILESYSTEM=1 `
    -s ASSERTIONS=1 `
    --preload-file assets
```

## Serve & Test Locally

WASM requires HTTP serving (file:// won't work):

```powershell
# Python
python -m http.server 8000

# Or with npx
npx serve .
```

Then open `http://localhost:8000` in a browser.

## Integration with Makefile

Add this target to your Makefile:

```makefile
EMCC := emcc
RAYLIB_WASM_DIR := raylib/install_wasm

wasm: src/main.cpp
	$(EMCC) -o index.html src/main.cpp \
		-I$(RAYLIB_WASM_DIR)/include \
		-L$(RAYLIB_WASM_DIR)/lib \
		-lraylib \
		-O2 \
		-s TOTAL_MEMORY=67108864 \
		-s FORCE_FILESYSTEM=1 \
		--preload-file assets
	@echo "WASM build complete: index.html + index.js + index.wasm"
```

Usage:
```powershell
make wasm
```

## Deployment

Upload these 3 files (plus the `.data` if using `--preload-file`) to any static host:

```
index.html
index.js
index.wasm
assets.data     (generated by --preload-file)
assets.js.map   (if present)
```

Works on GitHub Pages, Netlify, Vercel, itch.io, any static file server.

Emscripten's `index.html` uses WebGL2 (with WebGL1 fallback), which is supported in all modern browsers.

## Common Pitfalls

| Problem | Fix |
|---------|-----|
| `emcc not found` | Run `emsdk_env.ps1` first |
| White screen / no canvas | Check browser console for errors; WASM needs `Cross-Origin-Opener-Policy` for SharedArrayBuffer if using threads |
| Assets not loading | Verify `assets/` exists and `--preload-file assets` is passed; check the .data file is served alongside .wasm |
| "Cannot find -lraylib" | Raylib WASM build hasn't been run yet (see "Build Raylib for Web" above) |
| Memory exhausted | Increase `-s TOTAL_MEMORY` (must be power of 2: 134217728 for 128MB, etc.) |
| Game runs slow | Add `-O3` or `-s WASM=1` (already default); profile with browser dev tools |
